개발기술지원팀의 블로그입니다.

Guest users are not permitted to access this organization as per organization policy

Last Update: 2022-01-13 공유

Azure DevOps에 게스트 사용자를 이용해 Organization을 만들었을 때, 생성하는 것까지는 성공한 것으로 보이나 해당 Organization에 접근이 불가능합니다. (TF909091: Guest users are not permitted to access this organization as per organization policy)

TF909091 코드는 메시지 그대로 게스트 사용자가 정책에 의하여 조직에 접근할 수 없음을 의미합니다.
외부 사용자의 접근 허용 여부를 결정하는 “External Guest Access”라고 하는 옵션이 현재 기본 Off인 것으로 확인되는데, 보안적인 측면에서 게스트 사용자가 조직에 접근하지 못하는 상황입니다.

해당 문제를 피할 수 있는 방법은 현재 Guest인 사용자를 Member로 전환하고 브라우저 캐시를 완전히 삭제 후 로그인하실 수 있습니다.

<이미지 출처: https://docs.microsoft.com/ko-kr/azure/devops/organizations/accounts/add-external-user?view=azure-devops#prerequisites >

Azure AD 게스트 사용자를 Member로 전환하는 방법에 대해서는 문서 “Q: How can I convert an Azure AD guest into a member?” Access via Azure AD FAQs | Microsoft Docs 를 참고하셨으면 좋겠습니다.

※본 정보의 내용(첨부문서, 링크처 등을 포함)은 작성일 현재이며, 예고없이 변경될 수 있습니다.

최근 포스트

  • Microsoft Edge IE Mode 정책 적용에 대한 시점 의견
  • Edge 브라우저는 기본적으로 페이지 오픈을 '새탭'으로 열립니다.
  • Microsoft Edge IE Mode 구성 방법
  • Microsoft Edge IE모드 설정 시 IE모드에서 Redirection이 되면서 SSO(Single Sign On)기능 동작 실패
  • Microsoft Edge에서 제공하는 '확장 기능을 사용하여 브라우저 개인 설정' 페이지(edge://extensions/)에서 '압축 풀린 파일 로드'를 클릭했을 때 디렉터리 선택 창이 표시 안되는 이슈
