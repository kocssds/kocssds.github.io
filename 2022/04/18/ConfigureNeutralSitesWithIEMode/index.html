Microsoft Edge IE모드 설정 시 IE모드에서 Redirection이 되면서 SSO(Single Sign On)기능 동작 실패

Last Update: 2022-04-18 공유

Microsoft Edge IE모드 설정 시 IE모드에서 Redirection이 되면서 SSO(Single Sign On)기능 동작 실패를 경함한다면, 중립사이트 구성에 대해서 고려해보실 수 있습니다.

IE모드 구성시 SSO와의 원활한 통신을 위해서 다음을 확인해 주시기 바랍니다.

IE 모드 사이트 리스트에 SSO 관련 도메인에 대한 처리(중립 사이트 구성)

중립 사이트(Neutral Site) 구성

IE 모드가 제대로 작동하려면 인증/SSO(Single Sign-On) 서버를 중립 사이트로 명시적으로 구성해야 합니다. 그렇지 않으면 IE 모드 페이지가 Microsoft Edge로 리디렉션을 시도하여 인증에 실패할 수 있습니다.
중립 사이트는 탐색이 시작된 브라우저(Microsoft Edge 또는 IE 모드)를 사용합니다. 중립 사이트를 구성하면 최신 및 기존 인증 서버를 사용하는 모든 응용 프로그램이 계속 작동합니다.
엔터프라이즈 모드 사이트 목록 관리자 도구에서 열기 드롭다운을 ‘None’으로 설정하거나 사이트 목록 XML을 직접 업데이트하여 중립 사이트를 구성할 수 있습니다.

예시

  • 엔터프라이즈 모드 사이트 목록 관리자 도구에서 설정

  • XML 직접 수정 그러므로, 아래와 같은 고려사항이 있을 수 있습니다.
  1. 이와 같이 IE 모드를 구성하는 과정에서 SSO 인증과 같은 도메인에 경우 중립 사이트(Neutral Site)로 구성이 되어 있어야 하며,
  2. 필요에 따라 추가적으로 필요 시 Cookie가 Edge에서 IE 모드로 공유될 수 있도록 Shared-Cookie를 구성해야 할 수도 있습니다.
